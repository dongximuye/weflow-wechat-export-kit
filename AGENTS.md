# WeFlow 微信聊天导出交付项目规则

## 项目目标

为非技术同事提供可复现的 WeFlow 安装、微信聊天导出、完整性验收和 Codex 分析流程。

## 固定路线

- 使用 WeFlow 4.5.1 现有导出能力。
- 推荐 ChatLab JSONL，并保留完整媒体目录。
- 不继续开发数据库解密、密钥提取、注入或 Hook 方案。
- Skill、教程和启动提示词必须面向非技术同事。

## 安全边界

- 不把密钥、聊天记录、数据库或真实导出样本提交到 GitHub。
- 不在教程中展示未打码的账号、群名、联系人或聊天正文。
- 安装器只作为版本化发布附件，不进入 Git 历史。
- 公布第三方安装器前确认再分发授权，并披露哈希值与签名状态。

## 验收

- Skill 通过官方 `quick_validate.py`。
- 验收工具在真实 ChatLab JSONL 样本上通过，并能发现损坏行和缺失媒体。
- 完整启动提示词可引导 Codex完成安装、重启、能力说明、目标询问、导出和验收。
- 发布包不得包含真实聊天数据。

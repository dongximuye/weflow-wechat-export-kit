WeFlow 微信聊天导出截图教程

这是一份给同事看的“照着点”教程。先看本教程，再把同目录下的 给Codex的完整启动提示词.md 复制给 Codex。真正的工具包由 Codex 从 GitHub Release 下载。本 HTML 已内嵌截图，单独打开也能看到配图。

先别离开电脑：前 5 分钟 Codex 可能会请求访问 GitHub、下载文件、读取目录或安装 Skill。请等这些权限确认完，再去忙别的事情。

你需要准备

  • Windows 电脑，已安装并登录微信。
  • Codex 桌面版。
  • 空间充足的磁盘目录，用于安装和导出。
  • 要导出的聊天对象和时间范围。

不要上传这些

  • 微信数据库、密钥、原始导出包。
  • 包含孩子、家长、客户、手机号、地址、支付信息的原始内容。
  • 未经脱敏的图片、文件、语音和视频。

操作目录

1. 先让 Codex 接手 2. 安装 WeFlow 3. 找到微信数据库位置 4. 首次配置和密钥 5. 创建导出任务 6. 完成后交给 Codex 验收 7. 不建议让 Codex 代操安装

1先让 Codex 接手

打开同目录下的 给Codex的完整启动提示词.md，全文复制给 Codex。Codex 会先做权限预检，再去公共 GitHub 仓库下载工具包。

如果 Codex 一直停在“需要批准”或“等待权限”，通常不是坏了，而是在等你点确认。请先处理权限请求。

仓库地址是：https://github.com/dongximuye/weflow-wechat-export-kit

2安装 WeFlow

Codex 会优先从 GitHub Release 获取工具包。你也可以根据 Codex 指引手动打开安装器所在位置。

2.1 找到安装器

找到 WeFlow-4.5.1-x64-Setup.exe，双击打开。

2.2 选择安装方式

通常选择“仅为我安装”，然后点击“下一步”。如果电脑空间紧张，请选择 D 盘等空间较大的位置。

2.3 完成并启动

完成安装时保留“运行 WeFlow”勾选，点击“完成”。桌面通常也会出现快捷方式。

3从微信设置里找到数据库位置

WeFlow 首次配置时可能需要确认微信数据库或文件目录。不要凭感觉猜路径，按下面方式找：

  1. 打开 Windows 微信。
  2. 点击左下角“三横线 / 设置”。
  3. 进入“文件管理”或类似入口。
  4. 查看微信文件、数据所在位置。
  5. 回到 WeFlow，选择对应的数据库目录或上级微信数据目录。
这里只是定位目录。不要移动、删除、重命名微信数据库，也不要把数据库文件发给 Codex 或任何人。

4首次配置和自动获取密钥

如果电脑之前配置过 WeFlow，可能直接进入“解密密钥”页面；这是正常现象。

  1. 先保存微信里未完成的文字、文件传输或工作。
  2. 按 WeFlow 提示退出已经登录的微信。
  3. 点击“自动获取密钥”。
  4. 按界面指示操作，完成后返回主界面。
密钥不需要给 Codex 看。不要复制、截图、发送或写进文档。
聊天记录很多时，首次加载可能很慢，甚至短暂报错。先等待几分钟，点击刷新，再判断是否真的失败。不要一报错就重装。

5创建导出任务

5.1 进入导出页

点击左侧“导出”。

5.2 选择导出位置和会话

顶部选择空间充足的导出位置；在“私聊”或“群聊”中搜索目标会话，点击右侧“导出”。

5.3 选择格式、时间和媒体

文本格式建议选择 ChatLab JSONL，这是更适合 Codex 读取和完整性检查的结构化格式。时间范围按需要选择；媒体按分析需要勾选图片、文件、语音、视频、表情包。

5.4 设置文件大小和发送者名称

文件大小上限会影响能否导出大附件。无法预判时，优先保证完整性。群聊发送者名称一般可选“备注优先”或“群聊昵称优先”。

5.5 等任务真正完成

点击“创建导出任务”后，进入任务中心。任务仍在进行中时，不要移动导出目录，也不要说已经成功。

增量导出建议：不是第一次导出时，不要默认选择“全部时间”。先让 Codex 查看上一次 export_integrity_report.md 的末条时间，再选择“今天”或“从上次导出日期到今天”。如果 WeFlow 只能按日期选择，可以多导一天，再由 Codex 按消息 ID、时间和内容结构去重。已完成首次配置后，当天增量通常约 2–5 分钟/群，媒体较多约 5–15 分钟/群。

6导出完成后交给 Codex 验收

任务完成后，把完整导出目录路径发给 Codex。不要只给单个 JSONL，也不要改变 texts、images、file、videos 等目录的相对位置。

  • Codex 应检查聊天对象、时间范围、消息数量、首尾时间。
  • Codex 应检查 JSONL/文本文件、媒体目录、乱码、空文件、路径断裂。
  • Codex 最后应生成 export_integrity_report.md。
只有完整性检查通过后，才算“导出成功”。后续给 AI 分析时，优先使用脱敏后的文本、媒体清单和验收报告。

7不建议让 Codex 代操安装配置

默认做法是：Codex 负责读教程、判断下一步、提醒风险和验收结果；人负责点击 WeFlow、微信和系统弹窗。

不建议让 Codex 直接接管鼠标键盘完成安装配置。安装器、微信、密钥获取和系统弹窗会频繁切换窗口；如果这时使用其他软件，会影响 Codex 截屏和判断，容易点错、卡住或变慢。

只有在操作者愿意全程盯着电脑、不切换窗口、随时处理弹窗，并明确接受效率较低的情况下，才考虑让 Codex 辅助操作局部步骤。团队复现时，推荐人工点击 + Codex 指导。

WeFlow 是第三方工具。请仅在授权、合规、必要的范围内处理微信聊天记录。

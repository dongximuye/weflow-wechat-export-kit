# ExportWeChat Mac 导出与处理流程

本文件给 Codex 阅读。用户只需要提供目标聊天、导出目录和分析目的；不要要求用户自己研究本文。

## 适用范围

- 平台：macOS。
- 工具：ExportWeChat。
- 目标：把微信聊天导出结果验收为可供 AI 分析的本地资料包。
- 不负责：破解、提取密钥、读取微信原始数据库、读取系统钥匙串。

## 官方入口

- 官网：<https://www.aiyuziyou.com/>
- 会员/媒体能力：<https://www.aiyuziyou.com/pricing>

只从官方入口下载。不要去不明软件下载站。

## 安装与首次启动

1. 指导用户打开官网并下载 macOS 客户端。
2. 下载时页面可能提示一段终端代码。让用户复制官网显示的代码；Codex 不要凭空编写替代代码。
3. 下载完成后，让用户打开 ExportWeChat。
4. 如果 macOS 提示“大意为该文件已损坏”或无法打开，不要直接判断为文件真损坏，也不要反复重下。
5. 指导用户打开“终端”，粘贴官网提示的代码并回车。
6. 代码执行后，ExportWeChat 通常会自动启动；若没有，再让用户手动打开一次。
7. 仍无法启动时，区分下载不完整、macOS 安全限制、权限、芯片架构、应用故障。

## 会员与格式口径

用户实操反馈：

- 免费版只能导出 TXT。
- 需要 JSON、HTML、Markdown、PDF、Excel、CSV 等格式时，需要高级会员。
- 用户实操时看到高级会员费用为一年 39.9；价格以官网或应用内实时显示为准。

给 AI 分析时优先建议 JSON。若只有 TXT，按半结构化文本处理，并在报告中标注“格式能力受免费版限制”。

## 导出建议

1. 先确认导出对象：单聊、群聊或多个聊天。
2. 确认导出时间范围。
3. 优先导出 JSON；需要人工查看或媒体留档时，额外导出 HTML 或带资源目录的格式。
4. 是否能导出图片、视频、语音、附件，以套餐和实际目录检查为准，不提前承诺。
5. 导出完成后，让用户提供完整导出目录路径。

推荐本地结构：

```text
微信聊天导出_本地敏感/
├─ 00_待验收/
├─ 10_已验收原始包/
├─ 20_AI分析用脱敏版/
├─ 30_增量状态/
└─ 90_验收报告/
```

不要移动或删除原始导出包，除非用户明确确认。

## 验收流程

只读检查用户指定目录：

1. 目录是否存在、可读。
2. 文件数、目录数、总大小。
3. 空文件数量。
4. 是否存在 JSON、JSONL、HTML、Markdown、PDF、Excel、CSV、TXT。
5. 是否存在图片、文件、语音、视频、表情包目录或文件。
6. 是否疑似多个会话混在一起；如有，暂停确认。
7. 是否出现数据库、密钥、缓存等不应外发内容；只报告风险，不输出内容。

格式识别：

- JSON / JSONL：识别消息列表、消息 ID、发送者、时间、类型、正文、媒体路径。
- HTML：检查 `img src`、`video src`、`audio src`、`source src`、`a href` 是否指向包内文件。
- CSV / Excel：识别时间列、发送者列、正文列、类型列、ID 列。
- Markdown / TXT：按半结构化文本处理。
- PDF：只作为人工阅读版；除非没有其他格式，否则不作为主要机器分析源。

完整性检查至少包括：

1. 聊天对象是否匹配。
2. 实际首尾时间。
3. 消息数量。
4. 消息类型分布。
5. 重复消息 ID；无 ID 时用“时间 + 发送者 + 类型 + 内容哈希”估算重复。
6. 坏行、乱码、空文件。
7. 媒体引用是否存在。
8. 媒体路径是否越界指向导出目录之外。
9. 媒体文件是否为 0 字节。

如果文本通过但媒体未随包落盘，结论写成“文本通过，媒体未验收或未随包导出”，不能说完整导出成功。

## 增量策略

非首次导出时：

1. 读取上次 `export_integrity_report.md` 或状态文件。
2. 找到上次成功验收的末条消息时间和消息 ID。
3. 建议从上次日期当天开始多导一点。
4. 优先按消息 ID 去重；没有 ID 时用“时间 + 发送者 + 类型 + 内容哈希”去重。
5. 报告重复数量、保留数量、丢弃数量。
6. 更新下一次建议导出日期。

## 输出

必须生成：

```text
export_integrity_report.md
```

报告包含：

- 检查时间、导出对象、用户声明范围、实际首尾时间；
- 导出目录、文本/结构文件清单、媒体清单；
- 消息数量、类型分布、重复/缺失/空文件/乱码/坏行；
- 媒体路径检查结果；
- 是否通过验收；
- 可交给 AI 分析的文件；
- 必须留在本地的文件；
- 是否建议重新导出媒体；
- 增量导出的下一次建议起点。

可选生成：

```text
<聊天对象安全文件名>.export_state.json
```

状态文件只记录聊天对象、最后验收时间、末条时间、末条消息 ID、媒体状态、报告路径和下一次建议导出日期，不写聊天正文。

## 安全要求

- 不输出聊天正文、手机号、地址、身份证、支付信息、二维码内容或客户隐私。
- 不复制敏感媒体到脱敏目录，除非用户明确要求。
- 脱敏版必须保留可回查定位信息：消息序号、时间、发送者代号、消息 ID 或本地行号。

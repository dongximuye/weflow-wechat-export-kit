# ChatLab JSONL 说明

## 结构

ChatLab JSONL 每个非空行都是一个独立 JSON 对象。常见记录类型：

- `header`：格式和会话元数据。
- `member`：成员标识、昵称和头像。
- `message`：发送者、时间、消息类型、正文和原始消息标识。

媒体消息的 `content` 通常是相对于 `texts` 目录的路径，例如：

```text
../images/119_example.png
../file/pdf/605_example.pdf
```

必须以 JSONL 所在目录为基准解析路径，并确认解析后的文件仍处于同一导出包内。

## 分析顺序

1. 运行完整性检查工具。
2. 读取 `header`，确认会话身份。
3. 统计 `message` 数量、时间边界、发送者和消息类型。
4. 按用户问题确定日期范围或关键词，不盲目一次加载全部材料。
5. 遇到媒体路径时读取对应本地文件；无法解析的媒体明确标注。
6. 输出结论时保留时间、发送者和 `platformMessageId`，便于回查。

## 隐私边界

- 不在报告中展示微信密钥、数据库路径或无关账号标识。
- 不读取目标会话之外的导出文件，除非用户明确授权。
- 不把聊天内容发送到外部服务。
- 用户要求生成可转发报告时，先提醒检查姓名、手机号、群二维码和文件内容。

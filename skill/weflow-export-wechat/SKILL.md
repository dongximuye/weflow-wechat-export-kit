# WeFlow 微信聊天导出

## 总原则

- 使用 WeFlow 已有能力，不自行研究微信数据库解密、注入或 Hook。
- 将 WeFlow 界面操作留给用户，逐步指导；不要替用户自动退出、重启或登录微信。
- 不读取、复述、保存或上传解密密钥。
- 不上传聊天记录；只在用户明确指定的本地目录内读取和分析。
- 推荐导出 `ChatLab JSONL` 与所需媒体，保留 WeFlow 的完整目录结构。
- 完整性优先于格式美观。通过验收后再进行总结或分析。

## 工作流

### 1. 明确目标

先确认以下三项；每次只问必要问题：

1. 导出哪个单聊或群聊。
2. 导出全部时间还是指定日期范围。
3. 是否需要图片、文件、语音、视频和表情包。

若用户只想分析已经导出的资料包，直接进入“验收导出包”。

### 2. 检查 WeFlow

检查用户提供的安装包或 WeFlow 安装状态。未安装时，打开并遵循 [WeFlow操作教程.md](references/WeFlow操作教程.md) 的“安装”部分逐步指导。

安装与首次配置需要用户本人操作。到“自动获取密钥”步骤时：

1. 提醒用户保存微信中未完成的工作。
2. 明确说明需要按 WeFlow 界面提示退出微信。
3. 等用户确认后再继续讲下一步。
4. 不要求用户把密钥发给 Codex，也不要截取或输出密钥。

若 WeFlow 需要确认微信数据库目录，指导用户从 Windows 微信左下角“设置”进入“文件管理”查看微信文件/数据库所在位置。只定位目录，不移动、不删除、不重命名、不上传原始数据库。

若数据库量大导致首次加载很慢或短暂报错，先让用户等待几分钟并刷新；排查路径、微信是否退出、权限和磁盘空间后，再判断是否需要重新配置。不要一报错就让用户重装。

### 3. 创建导出任务

按 [WeFlow操作教程.md](references/WeFlow操作教程.md) 指导用户：

1. 进入“导出”。
2. 选择空间充足的导出目录。
3. 找到目标会话并点击“导出”。
4. 选择 `ChatLab JSONL`。
5. 设置时间范围。
6. 勾选所需媒体类型。
7. 根据需要设置文件大小上限、语音转文字和发送者名称显示方式。
8. 创建任务并等待任务中心完成。

不要把“任务已创建”当作“导出完成”。必须等待任务状态完成并打开结果目录。

### 4. 验收导出包

定位导出目录中的 `texts/*.jsonl`。运行：

```powershell
python scripts/validate_weflow_export.py "导出目录"
```

需要保存机器可读报告时运行：

```powershell
python scripts/validate_weflow_export.py "导出目录" --json-report "验收报告.json"
```

通过标准：

- 至少存在一个 JSONL 文件。
- 每个非空行均能解析为 JSON。
- 每个消息对象具有发送者、时间、消息类型、正文和原始消息标识。
- 原始消息标识不存在重复。
- 所有相对媒体路径均位于导出包内且文件存在。

失败时报告文件名、行号和缺失路径，但不要输出聊天正文。让用户回到 WeFlow 重导对应时间段或媒体类型；不要自行修补原始导出包。

### 5. 分析聊天

验收通过后阅读 [ChatLab-JSONL说明.md](references/ChatLab-JSONL说明.md)。先向用户复述：

- 会话名称与时间边界。
- 消息总量和媒体总量。
- 是否存在验收警告。
- 用户希望解决的具体问题。

分析时保留发送者、时间和原始消息标识作为证据定位；引用图片或文件时使用 JSONL 中的相对路径解析本地文件。材料较大时按日期分批读取，每批结束记录边界，避免重复或遗漏。

## 停止条件

遇到以下情况立即暂停并说明：

- 用户要求 Codex 展示或传输密钥。
- 导出目录包含用户未授权的其他会话。
- 媒体路径指向导出目录之外。
- JSONL 大量损坏或任务尚未完成。
- 操作将退出、重启、登录微信或重新获取密钥，但用户尚未明确确认。
